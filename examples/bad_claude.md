## Code Style

- Always use 2 spaces for indentation
- Maximum line length is 80 characters
- Use single quotes for strings
- Always add trailing commas
- Never use semicolons
- Use camelCase for variables
- Use PascalCase for classes

## Commands

- Run `npm test` to run tests
- Run `npm run build` to build
- Run `npm run lint` to lint
- Run `npm run format` to format
- Run `npm run dev` to start dev server
- Run `npm run prod` to start prod server
- Run `npm run e2e` to run e2e tests
- Run `npm run unit` to run unit tests
- Run `npm run coverage` to check coverage

## Database

When creating new database schemas:
- Always use UUID for primary keys
- Always add created_at and updated_at timestamps
- Never use cascade deletes
- Always add indexes on foreign keys
- Use snake_case for column names

## API Design

- Always return JSON responses
- Always include status codes
- Never expose internal errors
- Always validate input
- Use REST conventions
- Include pagination for lists

## Testing

- Always write unit tests first
- Never mock the database

- Use descriptive test names
- Ensure 80% coverage minimum
